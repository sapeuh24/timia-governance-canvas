📋 Buenas Prácticas
Guía para proponer Pre-Diccionarios · TC-015 NextGen v2.7
3.1 Consideraciones Previas
🔍
Búsqueda Previa
Verificar en DATUM si el campo ya existe antes de proponer un nuevo Pre-Diccionario
🗄️
Base de Datos
Confirmar dominio, esquema y tabla destino antes de iniciar la nomenclatura
✏️
Diseño TC-015
Aplicar reglas de nomenclatura NextGen v2.7 desde el inicio para evitar rechazos
3.2 Reglas y Validaciones
📏 Reglas de Nomenclatura Física
▼
✓Usar solo letras minúsculas, números y guion bajo (_)
✓El nombre físico debe iniciar con el nombre del objeto o concepto de negocio
✓Agregar el sufijo correspondiente al final según el tipo de dato (TC-015)
✓Máximo 64 caracteres para el nombre físico
✓No usar caracteres especiales, tildes ni espacios
🏷️ Nombre Lógico y Descripción
▼
✓El nombre lógico debe ser descriptivo y en español, reflejando el significado de negocio
✓La descripción debe incluir la fuente origen, uso del campo y ejemplos de valores
✓Máximo 255 caracteres para la descripción
✓Indicar si el campo es calculado, derivado o proveniente de fuente legacy
